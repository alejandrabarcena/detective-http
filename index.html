🕵️‍♀️ El detective del servidor

Este proyecto explora cómo un servidor responde a solicitudes HTTP desde el navegador. No se escribe código, solo se usan herramientas de inspección como DevTools y plataformas online. Se observa el comportamiento real de páginas web como una QA tester investigadora.

📚 Proyecto educativo

🌐 Sitios analizados

  • Deutsche Welle – Sitio de noticias y análisis internacional 🇩🇪
  • BBC Learning English – Plataforma educativa de inglés 🇬🇧
  • TV5Monde – Medio cultural francófono 🇫🇷

🛠️ Herramientas utilizadas

  • 🔍 DevTools del navegador (Network)
  • 📡 httpstatus.io
  • 📬 reqbin.com
  • 🎯 webhook.site

🔎 Qué se analizó

  • Códigos de estado HTTP: 200 (OK), 301 (Redirección), 404 (No encontrado)
  • Redirecciones automáticas visibles en DevTools
  • Encabezados HTTP: tipo de contenido, servidor, control de caché
  • Errores personalizados o estándar (404)
  • Comportamiento de caché en recarga: no-store, caché parcial, etc.

🧪 Simulaciones realizadas

  • URLs falsas para probar errores personalizados
  • Reload para observar headers como `Cache-Control` y `ETag`
  • Petición manual con `reqbin` para revisar headers completos

💡 Conocimientos obtenidos

  • Identificar códigos de estado HTTP y entender su significado
  • Leer encabezados HTTP y entender cómo responden los servidores
  • Usar herramientas como una tester web profesional
  • Analizar el performance y errores sin programar

🚀 Publicación

El proyecto fue publicado con GitHub Pages, con estructura HTML/CSS/JS. Se puede consultar en línea o como base de análisis para otras tareas de QA.
